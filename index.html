Summary

Scenario: Proactive Anomaly Detection in Microservices Using Multimodal Data
Microservice architectures have emerged as a dominant paradigm for constructing scalable and agile systems, where numerous independently deployable services collaborate to handle user requests. However, the intricate interactions between microservices and the rapid evolution of their technical stacks introduce significant challenges in maintaining system reliability. Proactive anomaly detection in such environments is critical yet daunting, as anomalies can manifest across heterogeneous multimodal data streams‚Äîincluding traces, logs, and metrics‚Äîeach capturing distinct aspects of system behavior (e.g., invocation topologies, operational events, and performance indicators).

Challenges: Up-to-date labels unavailable
This challenge arises from two factors. First, frequent changes in microservice systems render old label data obsolete and unusable. Second, the increasing intricacy of microservices interactions makes it extremely difficult to obtain accurate human labels, requiring substantial labeling costs.

Our propose method: ALARM
We therefore propose ALARM, a novel anomaly detection method that leverages existing labeled multimodal data from other systems to build effective anomaly detection models for the target microservice system. To mitigate the obstacles posed by complex relationships and inconsistency in data quality, on the one hand, ALARM introduces a user call-based graph fusion method to effectively model inter-modal and inter-microservices relationships. On the other hand, ALARM employs contrastive learning combined with virtual sample generation techniques to balance the two learning processes, increasing diversity from both aspects and thereby alleviating overfitting in modalities and domains.

Key Contributions

  • We propose a novel anomaly detection method, ALARM, for microservice systems. ALARM is tailored for the CADM task that tackle the challenges of heterogeneous modality data and scarce labels in a holistic way.
  • ALARM introduces a user call-based graph fusion approach to model complex inter-modal and inter-microservices relationships, improving transferable knowledge extraction. Contrastive learning is used to balance cross-modal and cross-domain learning, while virtual sample generation enhances diversity, addressing modal and domain overfitting.
  • Cross-system experiments on two microservice systems demonstrated the effectiveness of the ALARM approach.

Multi-modal data in microservices

Site Reliability Engineers (SREs) monitor microservices systems by relying on three key types of runtime data‚Äîtraces, logs, and metrics. Traces reveal service invocation topologies as tree-like call graphs with unique trace IDs and span IDs, documenting response times and status codes to enhance system transparency; logs are semi-structured textual outputs containing timestamps, log levels, and operational event details for inferring system state transitions; metrics serve as performance indicators (e.g., CPU utilization, memory consumption) collected at intervals into multi-dimensional time series, reflecting temporal fluctuations in system performance. Each modality embodies distinct structural characteristics and encapsulates spatial, temporal, and semantic information, and SREs integrate them to comprehensively assess the operational health and performance of microservices systems.

Main Ideas

ALARM includes three main phases: Feature Extraction, Call Graph-based Fusion and Multimodal Transfer Learning.
  • Feature Extraction: Transferable anomaly-relevant information (e.g., log semantics, metric patterns, trace response times/status codes) is extracted from heterogeneous modalities (traces, logs, metrics) and mapped into a unified feature space via modality-specific encoders.
  • Call Graph-based Fusion: A call graph-based strategy models complex inter-modal relationships by organizing multimodal data (generated under the same user call) into graph embeddings that preserve service invocation topologies and operational context. A Graph Autoencoder (GAE) fuses these embeddings into a unified vector, capturing cross-modal dependencies while enabling reversible decoding to retain intra-modal details.
  • Multimodal Transfer Learning: Contrastive learning promotes cross-modal/cross-domain information interaction to balance learning processes, while virtual sample generation enhances modal/domain diversity, mitigating overfitting.
